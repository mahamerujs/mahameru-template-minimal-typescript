# MahameruJS Minimal TypeScript Template

Minimal TypeScript template for building applications with MahameruJS. This template includes a welcome route, an in-memory projects API, shared error handling, and npm scripts for development, building, and running the application.

For easier setup, you can use the [create-mahameru](https://github.com/bintan/create-mahameru) command to scaffold a new project, or run:

```bash
npm create mahameru@latest
```

Then choose the "Minimal TypeScript" template.

## Manual Setup

### Prerequisites

Make sure you have installed:

- Node.js
- npm

Global Dependencies:
- [@mahameru/cli](https://www.npmjs.com/package/@mahameru/cli)

Dependencies:

- [mahameru](https://www.npmjs.com/package/mahameru)

Development Dependencies:

- [@types/node](https://www.npmjs.com/package/@types/node)
- [typescript](https://www.npmjs.com/package/typescript)

## Installation
Install [@mahameru/cli](https://www.npmjs.com/package/@mahameru/cli) as global dependencies:

```bash
npm install -g @mahameru/cli
```

Install project dependencies:

```bash
npm install mahameru
```

```bash
npm install -D @types/node typescript
```

## Running the Project

When running the project in developement mode or production mode, you need to install [@mahameru/cli](https://www.npmjs.com/package/@mahameru/cli) as global by running:

```bash
npm install -g @mahameru/cli
```
* Note: Some user may need administrator permission to install global packages

Run in development mode:

```bash
npm run dev
```

Build the project for production:

```bash
npm run build
```

For run in production mode, you need to open new terminal and run:
```bash
mahameru pm start
```
to start the MahameruJS Process Manager, then you can run:
```bash
cd .mahameru
```
and
```bash
npm start
```
on your project directory.

Notes:

- For detail commands of MahameruJS Process Manager, run `mahameru pm --help`

## Project Structure

```text
src/
  error.ts
  modules/
    projects/
      controller.ts
      service.ts
  routes/
    route.ts
    projects/
      route.ts
      [id]/
        route.ts
package.json
tsconfig.json
```

- `src/routes/route.ts` defines the root welcome endpoint.
- `src/routes/projects/route.ts` defines collection endpoints for projects.
- `src/routes/projects/[id]/route.ts` defines detail endpoints for a single project.
- `src/modules/projects/controller.ts` handles request and response flow for project routes.
- `src/modules/projects/service.ts` stores and mutates in-memory project data.
- `src/error.ts` converts thrown errors into JSON responses.

## API Usage

Use `<BASE_URL>` as the base URL where your MahameruJS app is running.

Project data is stored in memory, so it resets when the application restarts.

### GET /

Returns the welcome response.

```bash
curl <BASE_URL>/
```

Example response:

```json
{
  "success": true,
  "message": "Welcome to MahameruJS!"
}
```

### GET /projects

Returns all projects.

```bash
curl <BASE_URL>/projects
```

Example response:

```json
{
  "success": true,
  "data": []
}
```

### POST /projects

Creates a project. The `id` is generated by the service.

```bash
curl -X POST <BASE_URL>/projects \
  -H "Content-Type: application/json" \
  -d "{\"name\":\"Website Redesign\"}"
```

Example response:

```json
{
  "success": true,
  "data": {
    "id": 1,
    "name": "Website Redesign"
  }
}
```

### GET /projects/:id

Returns one project by ID.

```bash
curl <BASE_URL>/projects/1
```

Example response:

```json
{
  "success": true,
  "data": {
    "id": 1,
    "name": "Website Redesign"
  }
}
```

If the project does not exist, `src/error.ts` returns a 400 response:

```json
{
  "success": false,
  "message": "Project with id 1 not found"
}
```

### PATCH /projects/:id

Updates a project by ID.

```bash
curl -X PATCH <BASE_URL>/projects/1 \
  -H "Content-Type: application/json" \
  -d "{\"name\":\"Updated Website Redesign\"}"
```

Example response:

```json
{
  "success": true,
  "data": {
    "name": "Updated Website Redesign"
  }
}
```

### PUT /projects/:id

Uses the same update behavior as `PATCH /projects/:id`.

```bash
curl -X PUT <BASE_URL>/projects/1 \
  -H "Content-Type: application/json" \
  -d "{\"name\":\"Updated Website Redesign\"}"
```

Example response:

```json
{
  "success": true,
  "data": {
    "name": "Updated Website Redesign"
  }
}
```

### DELETE /projects/:id

Deletes a project by ID.

```bash
curl -X DELETE <BASE_URL>/projects/1
```

Example response:

```json
{
  "success": true
}
```

## Error Responses

Thrown `Error` instances are handled by `src/error.ts` and returned as JSON with HTTP status `400`.

```json
{
  "success": false,
  "message": "Project already exists"
}
```
