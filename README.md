# MahameruJS Minimal TypeScript Template

Minimal TypeScript template for building applications with MahameruJS. This template includes a welcome route, an in-memory projects API, shared error handling, and npm scripts for development, building, and running the application.

For easier setup, you can use the [create-mahameru](https://github.com/bintan/create-mahameru) command to scaffold a new project, or run:

```bash
npm create mahameru@latest
```

Then choose the **Minimal TypeScript** template.

## Manual Setup

### Prerequisites

Make sure you have installed:

- Node.js
- npm

Dependencies:

- [mahameru](https://www.npmjs.com/package/mahameru)

Development Dependencies:

- [@types/node](https://www.npmjs.com/package/@types/node)
- [typescript](https://www.npmjs.com/package/typescript)

## Installation

Install project dependencies:

```bash
npm install
```

## Create New Module
Create your first module by running:

```bash
npm run generate
```
then follow the instruction. For this example let's assume we create module named `project`.

## Running the Project

Run in development mode:

```bash
npm run dev
```

Build the project for production:

```bash
npm run build
```

Run production mode:
```bash
npm start
```

## Project Structure

```text
src/
  error.ts
  modules/
    project/
      controller.ts
      service.ts
  routes/
    route.ts
    project/
      route.ts
      [id]/
        route.ts
mahameru.config.ts
package.json
tsconfig.json
```

- `src/routes/route.ts` defines the root welcome endpoint.
- `src/routes/project/route.ts` defines collection endpoints for project.
- `src/routes/project/[id]/route.ts` defines detail endpoints for a single project.
- `src/modules/project/controller.ts` handles request and response flow for project routes.
- `src/modules/project/service.ts` stores and mutates in-memory project data.
- `src/error.ts` converts thrown errors into JSON responses.
- `mahameru.config.ts` define your mahameru config.

## API Usage

Use `<BASE_URL>` as the base URL where your MahameruJS app is running.

Project data is stored in memory, so it resets when the application restarts.

### GET /

Returns the welcome response.

```bash
curl <BASE_URL>/
```

Example response:

```json
{
  "success": true,
  "message": "Welcome to MahameruJS!"
}
```

### GET /project

Returns all project.

```bash
curl <BASE_URL>/project
```

Example response:

```json
{
  "success": true,
  "data": []
}
```

### POST /project

Creates a project. The `id` is generated by the service.

```bash
curl -X POST <BASE_URL>/project \
  -H "Content-Type: application/json" \
  -d "{\"name\":\"Website Redesign\"}"
```

Example response:

```json
{
  "success": true,
  "data": {
    "id": 1,
    "name": "Website Redesign"
  }
}
```

### GET /project/:id

Returns one project by ID.

```bash
curl <BASE_URL>/project/1
```

Example response:

```json
{
  "success": true,
  "data": {
    "id": 1,
    "name": "Website Redesign"
  }
}
```

If the project does not exist, `src/error.ts` returns a 400 response:

```json
{
  "success": false,
  "message": "Project with id 1 not found"
}
```

### PATCH /project/:id

Updates a project by ID.

```bash
curl -X PATCH <BASE_URL>/project/1 \
  -H "Content-Type: application/json" \
  -d "{\"name\":\"Updated Website Redesign\"}"
```

Example response:

```json
{
  "success": true,
  "data": {
    "name": "Updated Website Redesign"
  }
}
```

### PUT /project/:id

Uses the same update behavior as `PATCH /project/:id`.

```bash
curl -X PUT <BASE_URL>/project/1 \
  -H "Content-Type: application/json" \
  -d "{\"name\":\"Updated Website Redesign\"}"
```

Example response:

```json
{
  "success": true,
  "data": {
    "name": "Updated Website Redesign"
  }
}
```

### DELETE /project/:id

Deletes a project by ID.

```bash
curl -X DELETE <BASE_URL>/project/1
```

Example response:

```json
{
  "success": true
}
```

## Error Responses

Thrown `Error` instances are handled by `src/error.ts` and returned as JSON with HTTP status `400`.

```json
{
  "success": false,
  "message": "Project already exists"
}
```
